Concepts & Examples (from PPT ML Sections)

Definition: Machine Learning (ML) is a field of computer science that studies algorithms and techniques for automating solutions to complex problems that are hard to program using conventional methods.

ML focuses on creating predictive models that learn from data and generalize to unseen data. The larger the dataset, the more accurate they typically become.

Conventional Programming vs ML:

  • Conventional Programming: You provide Rules and Input Data. The computer follows the rules to produce the Output Data. (You tell the computer what to do).
  • Machine Learning: You provide Input Data and Output Data (examples). The ML algorithm finds the Rules (the relationship) between them. It can then predict the output for new input data. (You give it examples, and the computer figures out what to do).

Example 1: Summing two integers (Slide 8)

  • Conventional: Input X, Y; Rule Z = X + Y; Output Z. Simple to program.
  • ML: Input data: [(1,3), (4,7), (8,12), ...]; Output data: [(4), (11), (20), ...]. ML algorithm finds the rule (addition). Less efficient for such simple, rule-based problems.

Example 2: Image Recognition (Slide 9)

Given an image, decide if it's a cat or a dog.

  • By Conventional Programming: Extremely difficult. You would need explicit rules for every possible pixel pattern, lighting condition, angle, breed, etc. - practically impossible. ✗
  • By Machine Learning: Possible. Train a model on many labeled images of cats and dogs. The model learns features and patterns to distinguish them. ✓

Example: Given a picture of a handwritten digit, determine which symbol it represents (e.g., '2'). This is also possible with ML, but very hard with conventional programming.

Dramatic progress in ML since the mid-2000s is attributed to:

  • Availability of large amounts of data (e.g., images online).
  • Availability of large amounts of compute power, supported by large memory and storage space.
  • Improved algorithms optimized for large datasets.

Advancements in ML include:

  • Speech Recognition: Recognize speech and convert to text.
  • Language Translation: Understand and form language constructs without formal grammar training, with huge accuracy increases.
  • Driverless Vehicles: Navigate vehicles without human intervention.

Artificial Intelligence (AI): Broadest field. Goal is to create systems performing tasks requiring human intelligence (thinking, decision-making, problem-solving).

Machine Learning (ML): A subset of AI. Focuses on systems that *learn from data* and improve over time without explicit programming for every task. Goal: Build algorithms learning from data to make predictions/decisions.

Deep Learning (DL): A subset of ML. Uses artificial neural networks with multiple layers to model complex data patterns.

Other AI techniques exist that are *not* based on ML, e.g., Expert Systems, Game Playing (traditional), Traditional Computer Vision, Traditional NLP.

Venn diagram: AI is the outer circle, ML is a circle inside AI, and DL is a circle inside ML.

Labeled Data

Data where each instance has an assigned characteristic, category, or attribute (a "label"). Used in supervised learning.

Example: A photo explicitly tagged as "cat" or "dog".

Unlabeled Data

Data without predefined labels or categories. Used in unsupervised learning.

Example: A collection of photos without any tags; text documents needing to be grouped by topic.

Training Set

The portion of data used to train the machine learning model.

Testing Set

New data that the model has *not* seen during training, used to evaluate how well the model generalizes to real-world data.

Overfitting

Happens when a model learns the training data too well (memorizes it) but fails to perform accurately on new, unseen data. Often occurs if the model is too complex for the amount of training data.

Underfitting

Happens when a model is too simple to capture the underlying pattern in the data. It performs poorly on both the training data and new, unseen data.

Machine Learning is typically categorized into three main types:

  1. Supervised Learning:
    Used with labeled data (input-output pairs). The goal is to learn a mapping function to predict specific outcomes for new inputs.
    Includes: Classification, Regression.
  2. Unsupervised Learning:
    Used with unlabeled data (only input, no specific output provided). The goal is to discover inherent structures, patterns, or relationships.
    Includes: Clustering, Dimensionality Reduction.
  3. Reinforcement Learning:
    An agent learns to make sequences of decisions by interacting with an environment and receiving rewards or penalties. The goal is to maximize cumulative reward.

An agent interacts with an environment. The agent observes the state, chooses an action, and receives a reward.

Goal: Find the best action for each state to maximize cumulative reward.

How it Works:

  1. Agent observes the environment's state.
  2. Agent takes an action.
  3. Environment provides a reward (positive for good action, negative for bad).
  4. Agent learns which actions lead to higher rewards.
  5. Repeat to improve strategy over time.

Example: Robotics applications learning to walk.

Diagram: Agent takes Action -> Environment changes State and gives Reward -> Agent observes State and Reward.

Within supervised learning, the two main task types are distinguished by the type of output they predict:

  • Classification: Predicts a discrete category or class label (e.g., "cat" or "dog", "spam" or "not spam", "class A", "class B", "class C").
  • Regression: Predicts a continuous numerical value (e.g., house price, temperature, stock value).

Classification: Points grouped into distinct categories.

Regression: A line fitted through scattered points.

Classification uses labeled training data to learn a classifier model, which then assigns new input data to predefined categories.

Classifiers can be:

  • Binary Classifier: Identifies input as belonging to one of two output categories.
  • Multi-class Classifier: Identifies input as belonging to one of more than two output categories.

A common tool to evaluate the performance of binary classification models.

Objective: Evaluate performance.

How to Plot: A graph with:

  • X-axis: False Positive Rate (FPR)
  • Y-axis: True Positive Rate (TPR) (also known as Recall)

Formulas:

  • FPR = FP / (FP + TN): Ratio of actual negative instances incorrectly classified as positive.
  • TPR = TP / (TP + FN): Ratio of actual positive instances correctly classified as positive.

Types of ROC curves:

  • Diagonal ROC curve: Means the model performs like random guessing. For every correct positive, there's a false one (TP ≈ FP, TN ≈ FN). It's a bad model, cannot tell classes apart.
  • L-shaped ROC curve: Means the curve rises quickly toward the top-left corner. Model makes many correct predictions (True Positives are high, False Positives are low). It's a great model, separates classes well.

Diagonal ROC Curve: Straight line from (0,0) to (1,1).

L-shaped ROC Curve: Curve goes sharply up towards (0,1) then across.

A classification algorithm used for both binary and multi-class classification.

Examples: Classify emails as spam/ordinary; classify Iris flowers.

In Binary Classification:

Uses the Sigmoid function to compute the estimated output (predicted probability ŷ).

Formula: ŷ = 1 / (1 + e-z)

Where z is a linear combination of features X: z = b0 + b1x1 + b2x2 + ... + bnxn

The model's performance is evaluated using a loss function during training.

Loss function for binary classification: L(y, ŷ) = -[y * log(ŷ) + (1 - y) * log(1 - ŷ)]

Where y is the true label (0 or 1), and ŷ is the predicted probability.

In Multi-class Classification:

Uses the Softmax function to compute the probabilities for each class.

How the Algorithm Works (Training - Binary Example from COVID Diagnosis):

  1. Initialize the coefficients (b0, b1, ..., bn) in the sigmoid function (e.g., randomly).
  2. Take the first data instance (e.g., blood analysis features for patient 1, X).
  3. Compute the predicted output (ŷ) using the sigmoid function with the current coefficients and the instance's features.
  4. Compute the error between the predicted output (ŷ) and the actual output (y, the known diagnosis) using the loss function.
  5. Update the coefficients of the sigmoid function to minimize the error using an optimization algorithm like Gradient Descent.
  6. Repeat steps 3-5 for the second instance, and continue until all instances in the training set have been processed (completing one iteration or epoch).
  7. Perform more iterations (epochs) over the entire training set until the model reaches convergence and high accuracy is achieved.
