Concepts & Examples

Machine Learning (ML): A field of computer science studying algorithms and techniques for automating solutions to complex problems hard to program conventionally (S7).

Goal: Create predictive models that learn from data and generalize to unseen data. Accuracy improves with more data (S7).

Conventional Programming

You tell the computer what to do by providing a detailed, fixed set of steps or rules.

  • Design program logic (Rules).
  • Implement rules in code.
  • Input -> Apply Rules -> Output.

Machine Learning

You give the computer examples (Input-Output pairs), and it figures out the rules itself.

  • Provide Input data + Output data (Examples).
  • ML Algorithm finds relation (Rules).
  • New Input -> Apply Learned Rules -> Predicted Output.

ML enables solving problems like image recognition, language translation, and navigating vehicles, which are difficult with rigid conventional programming (S9, S10).

Recent Progress (Mid-2000s onwards) driven by: Availability of large data (Internet), Large compute power/memory, Improved algorithms (S10).

Artificial Intelligence (AI): The broad goal to create systems performing tasks requiring human intelligence.

Machine Learning (ML): A subset of AI focusing on systems that learn from data and improve over time without explicit programming.

Deep Learning (DL): A subset of ML using multi-layered artificial neural networks for complex pattern modeling.

Note: Not all AI techniques are based on ML (e.g., Expert Systems, traditional Game Playing, traditional Computer Vision/NLP) (S11).

Labeled vs. Unlabeled Data (S12)

  • Labeled Data: Data with assigned categories, characteristics, or known output attributes (e.g., photos tagged "cat" or "dog").
  • Unlabeled Data: Data without predefined labels or outputs (e.g., photos without tags).

Examples: (S12 Image described)

Training Set & Testing Set (S13)

  • Training Set: Data used to train the ML model.
  • Testing Set: New data the model has never seen, used to evaluate how well the model generalizes to real-world data.

Overfitting & Underfitting (S13)

  • Overfitting: Model learns training data (including noise) too well, fails to generalize to new data. Happens when the model is too complex for the data.
  • Underfitting: Model is too simple, fails to learn the underlying pattern in the data. Performs poorly on both training and test data.

ML is broadly categorized into three types (S14, S15):

  1. Supervised Learning: Uses labeled data to predict a specific outcome. (Examples: Classification, Regression)
  2. Unsupervised Learning: Uses unlabeled data to discover structures, patterns, or relationships. (Examples: Clustering, Dimensionality Reduction, Association Rule Mining)
  3. Reinforcement Learning: An agent learns by interacting with an environment and receiving rewards/penalties.

Classification (S19, S20)

Predicts a discrete category or class label.

Requires training data with predefined classes.

  • Binary Classification: Two output categories (S20, S21). Ex: Spam/Not Spam, Covid Positive/Negative.
  • Multi-class Classification: More than two output categories (S20, S22). Ex: Iris flower types (setosa, virginica, versicolor), email types.

Regression (S19)

Predicts a continuous numerical value.

Ex: Predicting house prices, stock values, temperature.

The ROC Curve (Receiver Operating Characteristic) is a common tool to evaluate the performance of binary classification models (S23).

It plots the True Positive Rate (TPR) on the Y-axis against the False Positive Rate (FPR) on the X-axis at various classification thresholds (S23).

  • TPR = TP / (TP + FN) (Recall)
  • FPR = FP / (FP + TN)

Diagonal ROC Curve

Represents random guessing. The model cannot effectively separate the classes.

L-shaped ROC Curve

Indicates a good model. Rises quickly towards the top-left corner (high TPR, low FPR), meaning it separates classes well.

Logistic Regression is a classification algorithm used for binary or multi-class classification (S25, S26).

For Binary Classification (S26, S27):

Uses the Sigmoid function to output a value between 0 and 1, representing the probability of belonging to the positive class.

Predicted output: ŷ = 1 / (1 + e⁻ᶻ)

Where z is a linear combination of input features X and coefficients (weights) b: z = b₀ + b₁x₁ + b₂x₂ + ... + bnxn

Model performance is evaluated using a Loss Function. For binary classification, the common loss is:

L(y, ŷ) = -[y * log(ŷ) + (1 - y) * log(1 - ŷ)]

Where y is the true label (0 or 1) and ŷ is the predicted probability.

The model learns by minimizing this loss function, typically using an optimization algorithm like Gradient Descent (S27).

For Multi-class Classification (S26):

Uses the Softmax function instead of Sigmoid to output a probability distribution over all classes.

How the Algorithm Works (Conceptual Steps - S27):

  1. Initialize the coefficients (b values) for the sigmoid function.
  2. Input the first data instance (features X).
  3. Compute the predicted output ŷ using the current coefficients and sigmoid function.
  4. Compute the error between the predicted output (ŷ) and the actual output (y) using the loss function.
  5. Update the coefficients using an optimization algorithm (like Gradient Descent) to minimize the calculated error.
  6. Input the next instance and repeat steps 3-5 for all instances (this completes one iteration/epoch).
  7. Repeat multiple iterations until the model converges and achieves high accuracy.

Unsupervised learning is used when there is no labeled data and no specific output to predict (S17).

Goal: Let the machine discover structures, patterns, or relationships in the data on its own (S17).

Helpful when you don't know the categories/groups in advance or want to group similar items (S17).

Techniques look for similarities to group data or reduce dimensions (S17).

Applications: Customer segmentation, organizing documents by topic, reducing data for visualization (S17).

Involves an agent interacting with an environment to learn optimal actions (S18).

The agent observes the environment's state, chooses an action, and receives a reward or penalty (S18).

Goal: Find the best action for each state to maximize cumulative reward (S18).

  1. Agent observes the environment's state.
  2. Agent takes an action.
  3. Environment provides a reward (positive or negative).
  4. Agent learns which actions lead to higher rewards.
  5. Repeat the process, agent tries to improve its strategy over time.

Example Application: Robotics (learning how to walk) (S18).
